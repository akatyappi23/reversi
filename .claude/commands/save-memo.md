直前の会話で出力した実装解説の内容を、`docs/memo/` ディレクトリに Markdown ファイルとして保存してください。

## メモのファイル名

- ユーザーの引数: `$ARGUMENTS`
- 引数がある場合: その内容をファイル名に使う（例: `ゲーム開始API` → `docs/memo/ゲーム開始API.md`）
- 引数がない場合: 解説内容から適切なトピック名を判断してファイル名にする

## メモのフォーマット

既存の `docs/memo/` 内のファイルと同じスタイルで書いてください:

- `# タイトル` から始める
- セクションを `## 見出し` で分ける
- セクション間を `---` で区切る
- コード例を含める
- Web初学者向けに丁寧に説明する
- 実装の解説だけでなく、使われている技術の背景知識もメモに含める
